# COrebound LAttence 玩家介面按鈕與操作總清單

本文件用途：

- 先把遊戲中所有可能的可按按鈕、可點擊區、可切換項目、可互動內容整理成總表
- 這一版**先不決定位置**
- 這一版**先不畫版面**
- 這一版的目的是幫後續 UI 擺放做母清單

本文件只做文字策劃，不涉及程式碼。

---

## 0. 使用方式

這份清單不是最後版面，而是「待分配元件表」。

之後可以這樣使用：

1. 先看這裡有沒有漏掉功能
2. 再把每個按鈕決定它是：
   - 永久顯示
   - 滑鼠停留才顯示
   - 接近物件才啟用
   - 只在某個介面中出現
3. 最後才去決定放在哪裡

---

## 1. 先分兩大類

為了不讓清單爆開，我先把所有操作分成兩大類：

### A. 戰鬥中操作

玩家在局內、角色還在移動時會用到的。

### B. 管理型操作

玩家打開圓盤管理介面、商店、合成台、圖鑑、結算頁時會用到的。

## 2. 戰鬥中操作總表

## 2.1 核心移動與戰鬥操作

這些不一定都是「按鈕圖示」，有些是輸入行為，但一樣要納入操作清單。

### 必備操作

- `移動`

## 6. 目前最像會成為主介面的操作核心

按照你現在的方向，我覺得未來最核心的操作不是傳統大片按鈕，而是這幾個：

1. `底部隊長頭像區`
2. `滑鼠停留展開圈層`
3. `點擊圓盤進入管理介面`
4. `在圓盤中選角色`
5. `在右側看立繪與細節`
6. `在同一介面裡切換背包 / 合成可用狀態`

也就是說，這套 UI 的靈魂不是「很多按鈕」，而是「一個核心圓盤帶出很多狀態」。

這點很棒，因為它很有你這個遊戲自己的臉。

## 7. 建議後續拆分順序

這份總清單做完後，下一步不要急著全部擺位置。
比較順的做法會是：

1. **先圈出第一批真的要上畫面的按鈕**
2. **再圈出滑鼠停留才出現的**
3. **再圈出接近物件才啟用的**
4. **最後才排版**

## 8. 下一輪最適合做的事

我建議下一輪可以從下面三個方向選一個：

1. `第一批必備按鈕縮表`
把這份總清單收斂成第一版真的要活著的按鈕。

2. `按鈕出現條件清單`
整理哪些按鈕是平常顯示、停留顯示、接近物件顯示、進介面才顯示。

3. `主畫面 HUD 專用按鈕清單`
只看戰鬥主畫面，不管商店、不管圖鑑，先把底部與局內那一層梳乾淨。
