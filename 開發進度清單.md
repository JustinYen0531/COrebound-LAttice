# COrebound LAttence 開發進度清單

本文件不是程式開發筆記，也不是單次 prompt 集。

它的用途是把整個專案重構成一份可以分階段執行的工作清單，方便你之後逐步下任務給不同 agent，並且判斷「這一階段是否已經做完，能不能進下一階段」。

---

## 1. 使用方式

每一個階段都固定看四件事：

1. 這一階段主要交給誰
2. 這一階段的核心目標是什麼
3. 這一階段有哪些小項目一定要完成
4. 什麼條件達成後，才可以進入下一階段

這樣做的好處是：

- 你可以很清楚知道現在該叫 agent 做什麼
- 不會提早把美術、音效、系統整個攪在一起
- 每次都能用「完成條件」來驗收，而不是感覺差不多

---

## 2. 職責分工

### 2.1 前半部

前半部以 `Coding Agent / AI Agent` 為主。

這一段重點是：

- 把遊戲的骨架搭起來
- 把主流程、HUD、管理介面、主畫面、互動邏輯做完整
- 把之後可以接圖、接音效、接資料的位置先留好

### 2.2 後半部

後半部以 `你自己 + 外部訂閱服務 / 生成工具 / 委託製作` 為主。

這一段重點是：

- 產出角色圖、怪物圖、世界圖、UI 素材
- 產出音樂、音效、影片、宣傳素材
- 逐步替換前期 placeholder，讓整體體驗成形

---

## 3. Agent 執行階段

## 階段一：規格定版與結構整理

### 主要負責

- `Coding Agent`

### 本階段目標

- 把目前所有已經定下來的玩法、主畫面、HUD、管理介面、互動介面規則收整成一致版本
- 把專案內的文件結構整理好，避免後續規格互相衝突
- 明確標記哪些是已定案，哪些是待討論

### 必須完成的小項目

- 主畫面 user flow 定版
- 正式遊玩畫面與管理介面切換規則定版
- HUD 核心構圖定版
- 隊長頭像展圈互動規則定版
- 左滑技能欄、右滑物品欄規則定版
- 管理介面內的資料夾分頁架構定版
- 互動資料夾的規則定版
- 主畫面按鈕與子按鈕清單定版
- 開發文件命名與用途整理完成

### 進入下一階段條件

- 所有主要介面都有文字規格
- 不再出現兩份文件互相打架的情況
- 已經能說清楚玩家從主畫面到戰鬥、再到管理頁的完整流程

---

## 階段二：核心遊玩骨架與戰鬥 MVP

### 主要負責

- `Coding Agent`

### 本階段目標

- 先把「可以玩」的最小版本建立起來
- 不求美術完整，但要能進場、移動、碰撞、存活、結算
- 建立時間進行中的正式遊玩畫面主循環

### 必須完成的小項目

- 角色進入戰局與出生流程
- 基本移動與鏡頭跟隨
- 基本敵人生成與刷新
- 基本攻擊、碰撞、受傷、死亡判定
- 生命值與能量值的實際系統
- 安全區 / 毒圈 / 收縮壓力機制
- 勝敗判定
- 戰局結束與返回流程
- Placeholder UI 能顯示生命值、能量值、冷卻狀態

### 進入下一階段條件

- 玩家可以從開始遊玩一路進到一場完整戰局
- 一場戰局可以正常開始、進行、結束
- 戰鬥核心數值至少能跑通，不會卡死在中途

---

## 階段三：HUD 操作層與戰鬥中可操作內容

### 主要負責

- `Coding Agent`

### 本階段目標

- 把你目前已經想好的 HUD 操作方式真正接到遊戲裡
- 讓玩家在戰鬥中能看懂主動技能、武器群組、週期技能、補給物件

### 必須完成的小項目

- 底部隊長頭像固定在主 HUD 核心位置
- 左側生命值條、右側能量值條配置完成
- 滑鼠停留隊長頭像後的展圈提示完成
- 最內圈、中圈、外圈分段展開邏輯完成
- 三圈各自的藍、紅、黃欄位結構建立
- 各圈有不同方向與不同速率的轉動表現
- 點擊圓圈任一區域可進入對應管理視圖
- 主動位移技能的正式冷卻顯示完成
- 武器群組的小圖示冷卻顯示完成
- 週期技能 / 自動施法的小圖示冷卻顯示完成
- 左滑展開技能欄完成
- 技能欄內可單獨關閉運作中的技能
- 右滑展開物品欄完成
- 物品欄可點按使用，也可拖曳到隊員身上
- 右側隊員狀態條與快速指派目標完成

### 進入下一階段條件

- HUD 不只是顯示資訊，而是真的能操作
- 左滑與右滑展開行為不互相衝突
- 玩家已能在戰鬥中完成技能管理與補給分配

---

## 階段四：管理介面與互動系統接線

### 主要負責

- `Coding Agent`

### 本階段目標

- 把管理介面做成完整的資訊與操作中心
- 把場上的互動物件接到管理介面中的互動資料夾

### 必須完成的小項目

- 管理介面開啟與關閉轉場
- 背景暗沉並聚焦中央圓盤的過程
- 小隊主頁的大圓盤放大檢視
- 點選圓盤成員後，左圓盤右立繪的配置完成
- 角色細節描述區完成
- 合成食譜顯示區完成
- 小隊成員狀態總覽完成
- 背包管理頁完成
- 互動資料夾完成
- 合成台、熔爐、雕像、商店、召喚等按鈕入口完成
- 未靠近互動物件時的「未開放 / 可嘗試操作」提示完成
- 靠近互動物件時的驚嘆號提示完成
- 點擊互動物件後自動跳到管理介面對應子頁
- 圖鑑資料夾完成
- 圖鑑下的怪物圖鑑、成員圖鑑、世界故事入口完成
- 地圖縮影按鈕與局內入口完成

### 進入下一階段條件

- 管理介面已能獨立承載小隊資訊、背包資訊、互動資訊、圖鑑資訊
- 場上互動物件都能正確導向各自的管理子頁
- 小隊圓盤頁與其他資料夾頁的分工清楚，不會混在一起

---

## 階段五：主畫面、存檔與遊戲外流程完整化

### 主要負責

- `Coding Agent`

### 本階段目標

- 把遊戲外層的入口做好，讓整個產品像一個完整遊戲，而不是只有戰場

### 必須完成的小項目

- 主畫面五到六個主按鈕配置完成
- 主按鈕點下後，右側彈出子按鈕的行為完成
- 再次點擊主按鈕時能收回子按鈕
- `開始遊玩` 內含 `New Game`、`Continue Game`、`多人連線 placeholder`
- `Continue Game` 的多存檔選擇流程完成
- `圖鑑` 入口可在非遊戲中查看內容
- `圖鑑` 下包含 `世界故事`
- `新手入門` 內含 `導覽` 與 `道場 / 沙盒`
- `遊玩記錄` 完成
- `遊玩記錄` 內含通關次數、使用角色、時間、成就
- `設定` 入口完成
- 音量、音效、基礎系統設定先做第一版

### 進入下一階段條件

- 玩家從開遊戲到進入遊玩、查圖鑑、看紀錄、進訓練都走得通
- 主畫面與遊戲內頁面的流程一致
- 存檔、讀檔、回到主畫面不會中斷主流程

---

## 階段六：資料接線、數值基底與替換接口

### 主要負責

- `Coding Agent`

### 本階段目標

- 幫後續素材、角色、怪物、裝備、音效預留乾淨接口
- 讓之後外部生成的資產可以一批一批接進來，而不是每次重做系統

### 必須完成的小項目

- 角色資料表結構完成
- 隊長資料、成員資料可獨立配置
- 武器群組資料表結構完成
- 週期技能 / 被動技能資料表結構完成
- 怪物與 Boss 資料表結構完成
- 掉落物與材料資料表結構完成
- 食譜 / 合成 / 升星資料結構完成
- 圖鑑資料可由統一格式讀取
- 音樂與音效掛載接口完成
- 立繪、頭像、圖示、影片等素材掛載接口完成
- Placeholder 可被真素材安全替換
- UI 不同尺寸素材的容錯規則建立

### 進入下一階段條件

- 後續新增內容時，不需要重寫整套系統
- 外部資產可以按批次逐步匯入
- 遊戲資料結構已經足以支撐擴充

---

## 4. 外部素材與服務階段

## 階段七：第一批素材生成

### 主要負責

- `你自己`
- `外部 AI 生圖 / 生成音樂 / 音效服務`

### 本階段目標

- 先把最能讓遊戲長出臉的素材做出來
- 這一批不是求全，而是求「一打開就知道這個遊戲是什麼」

### 第一批建議內容

- `4` 位隊長完整立繪
- `20` 位小隊成員的第一版立繪
- 至少 `24` 份角色頭像裁切
- `29` 種怪物 / Boss 的第一版主視覺方向
- `4` 張世界主氣氛圖
- 主畫面主要 UI 視覺零件
- HUD 核心零件
- 主畫面主題曲
- 前期探索音樂
- 主畫面與 HUD 核心音效

### 必須完成的小項目

- 每一類素材都有明確命名規則
- 每一類素材都有對應放進 `assets` 的位置
- 隊長、成員、怪物、世界主題的風格方向先一致
- 至少完成一輪「生成 -> 篩選 -> 保留可用版本」
- 主畫面與 HUD 已可替換第一批正式素材

### 進入下一階段條件

- 遊戲已經有第一批代表性視覺與聲音
- 不再只是 placeholder 方塊與空白音效
- 主要世界觀與角色辨識度已經建立

---

## 階段八：第二批素材生成

### 主要負責

- `你自己`
- `外部 AI 生圖 / 生成影片 / 生成音效服務`

### 本階段目標

- 把遊戲從「有臉」推進到「有生活感」
- 補齊互動物件、世界物件、材料圖示、操作演出素材

### 第二批建議內容

- 合成台、熔爐、雕像、商店、召喚裝置等互動物件
- 四大世界環境物件與障礙物
- `25` 種材料圖示
- 地板材質與世界裝飾
- 地圖縮影與世界預覽圖
- 管理介面資料夾、分頁、圖鑑零件
- 技能展示短片
- 世界展示短片
- 中後期戰鬥音樂
- 合成、互動、技能、物品音效

### 必須完成的小項目

- 各世界的互動物件有一致語言
- 背包、圖鑑、互動資料夾不再只靠文字撐場
- 音效已能覆蓋主要操作點
- 第二批素材接入後，管理介面的閱讀與辨識明顯提升

### 進入下一階段條件

- 玩家在主畫面、戰鬥、管理介面三邊都能感受到同一套世界觀
- 互動物件和資源物件不再抽象
- 主要操作已有足夠的聲畫回饋

---

## 階段九：第三批素材生成

### 主要負責

- `你自己`
- `外部 AI 影片 / 音樂 / 宣傳設計服務`

### 本階段目標

- 把遊戲推向正式發表感
- 補完高價值演出、宣傳資產與後期細節

### 第三批建議內容

- 最終 Boss 與世界守護者高完成版主視覺
- 勝利 / 失敗 / 通關結算專屬素材
- 宣傳圖、封面圖、商店圖
- 預告影片與主視覺短片
- 新手導覽影片
- 訓練道場示範影片
- 世界狂暴狀態的特殊演出素材
- 最終 Boss 音樂
- 勝利 / 失敗 / 成就 / 記錄翻看音效
- 風格變體版 UI 素材與宣傳版本圖

### 必須完成的小項目

- 商店頁或展示頁需要的資產已齊
- 遊戲內高重要度事件有專屬聲畫辨識
- 教學、展示、宣傳不需要再完全靠口頭說明

### 進入下一階段條件

- 遊戲已具備對外展示、測試招募、發布前整理的基礎
- 重要內容都有正式版資產可替換

---

## 階段十：整合驗收與最終打磨

### 主要負責

- `Coding Agent`
- `你自己`

### 本階段目標

- 把前面做好的系統與素材真正收束成一個可以展示、可以測試、可以持續擴充的完整版本

### 必須完成的小項目

- 所有第一批到第三批素材接線完成
- 壞掉的圖、音效、空連結、錯頁面全部修正
- 主畫面、正式遊玩、管理介面三大流程完整驗收
- 新手導覽與道場可正常進入
- 圖鑑、世界故事、遊玩記錄可正常閱讀
- 互動物件與管理介面子頁全部對得上
- 存檔、讀檔、成就、記錄能正確保存
- 重要操作的視覺與音效回饋完整
- 效能、讀取、素材尺寸與布局做基本優化

### 完成判定

- 遊戲已不是單點功能展示，而是完整體驗版本
- 可進行內部測試、外部展示、下一輪平衡調整

---

## 5. 建議執行順序

如果你之後要真的一段一段派工，我會建議照這個順序：

1. `階段一` 先把規格與文件對齊
2. `階段二` 做出能跑的戰鬥骨架
3. `階段三` 補 HUD 操作層
4. `階段四` 完成管理介面與互動系統
5. `階段五` 完成主畫面與遊戲外流程
6. `階段六` 做資料接線與素材接口
7. `階段七到九` 由你搭配外部服務逐批補素材
8. `階段十` 做整合驗收與最終打磨

---

## 6. 這份清單的核心價值

這份清單最重要的不是「列很多事情」。

而是讓你之後每次下任務時，都能用下面這種方式思考：

- 現在我們在哪一個階段
- 這一階段還缺哪些小項目
- 哪些做完才可以合理進下一步
- 這一步該交給 Coding Agent，還是交給外部生成服務

這樣整個專案會比較穩，不容易一邊做 UI、一邊又跳去做宣傳圖，最後全部都半套。
