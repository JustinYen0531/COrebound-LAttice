# COrebound LAttence 玩家使用介面 User Flow 方案

本文件只做文字策劃，不涉及程式碼、元件實作或技術細節。

目標：為 **COrebound LAttence** 規劃幾套從「最簡易」到「稍微有點花樣」的玩家介面流程，方便後續決定實作方向。

---

## 0. 設計原則

這款遊戲的資訊密度高，但玩家在對戰時不能被 UI 壓死，所以所有方案都遵守三件事：

1. **戰鬥中只看最重要的事**
玩家在局內最需要知道的是：
隊長安不安全、能量夠不夠、目前技能能不能打、附近互動點能不能用。

2. **編隊與養成資訊集中在非戰鬥畫面**
三層陣型、星級、家族、分支這些深度資訊，應該在備戰或停留互動畫面時展開。

3. **按鈕位置固定，不要漂移**
同類型按鈕盡量永遠在同一區。
例如：確認類在右下，返回類在左上，戰鬥主技能在右下或底部中央。

---

## 1. 方案總覽

### 方案 A：極簡直覺型
適合先做 MVP。

- 特點：按鈕少、資訊直接、學習成本最低
- 風格：像乾淨版生存競技 HUD
- 風險：特色感較弱，神祕圖騰感不夠強

### 方案 B：功能平衡型
適合正式第一版。

- 特點：兼顧可讀性與世界觀風格
- 風格：在易用基礎上加入同心圓與圖騰感
- 風險：資訊比 A 多，需要控好版面

### 方案 C：圖騰儀式型
適合做出記憶點。

- 特點：把很多介面都做成「儀式盤 / 同心圓 / 轉盤」
- 風格：辨識度最高，最有作品氣味
- 風險：若處理不好，會變帥但不耐打

---

## 2. 共通介面拆分

這三套方案都涵蓋以下 5 個玩家常用介面：

1. 主選單
2. 備戰與陣型編排
3. 局內戰鬥 HUD
4. 地圖互動介面
5. 對局結算介面

---

## 3. 方案 A：極簡直覺型

## 3.1 主選單

### 按鈕位置

- 畫面中央直式排列四顆大按鈕
- 左上：`設定`
- 右上：`玩家資料`
- 正中央：`開始對戰`
- 中央下方：`編排小隊`
- 再下方：`圖鑑`
- 最下方：`離開`

### User Flow

```text
開啟遊戲
  -> 主選單
    -> 點「開始對戰」
      -> 選擇隊長
        -> 確認小隊
          -> 進入匹配 / 開局

開啟遊戲
  -> 主選單
    -> 點「編排小隊」
      -> 調整陣型
        -> 儲存
          -> 回主選單
```

### 適合感受

- 很快進入遊戲
- 很少迷路
- 第一次玩的人最容易懂

## 3.2 備戰與陣型編排

### 按鈕位置

- 左側：角色清單
- 中央：三層同心圓陣型盤
- 右側：角色資訊卡
- 左上：`返回`
- 右下：`儲存編隊`
- 右上：`自動配置`

### 互動流程

```text
進入編排小隊
  -> 從左側點一名角色
    -> 角色資訊顯示在右側
      -> 拖到中央某一層
        -> 再點角色職責按鈕
          -> 設為 保護 / 支援 / 火力
            -> 按「儲存編隊」
```

### 角色職責按鈕位置

- 右側角色卡中段放三顆小按鈕：
  - `保護`
  - `支援`
  - `火力`

## 3.3 局內戰鬥 HUD

### 按鈕位置

- 左上：小地圖
- 左下：藥水與快捷道具
- 畫面下方中央：隊長能量條
- 右下：主動技能鍵
- 右側中段：5 個武器 / 家族技能狀態
- 畫面中央偏上：縮圈與危險提示

### User Flow

```text
進入對局
  -> 用 WASD / 搖桿移動
    -> 看下方中央能量條
      -> 能量夠時按右下主動技能
        -> 遭遇敵人
          -> 觀察右側技能狀態
            -> 靠走位與自動輸出作戰
              -> 血量低時按左下藥水
```

### 局內畫面重點

- 不開二級選單
- 商店、工作台、雕像都用靠近觸發
- 互動時只跳簡單彈窗

## 3.4 地圖互動介面

包含：雕像、熔爐、工作台、商店、寶箱。

### 按鈕位置

- 互動後跳出畫面中央面板
- 左側：投入材料或可選項目
- 右側：結果預覽
- 左下：`取消`
- 右下：`確認`

### User Flow 範例：商店

```text
靠近商店
  -> 按互動
    -> 跳出商店面板
      -> 左側選藥水
        -> 右側看價格與效果
          -> 按右下「購買」
            -> 回到戰場
```

### User Flow 範例：工作台

```text
靠近工作台
  -> 按互動
    -> 選擇想升星的角色
      -> 顯示需要材料
        -> 按右下「升星」
          -> 成功提示
            -> 關閉面板
```

## 3.5 對局結算介面

### 按鈕位置

- 畫面中央：名次、存活時間、擊殺數
- 左下：`回大廳`
- 右下：`再來一場`
- 右上：`戰報明細`

### User Flow

```text
對局結束
  -> 顯示名次與核心數據
    -> 玩家可選
      -> 「再來一場」
      -> 或「回大廳」
      -> 或打開「戰報明細」
```

## 4. 方案 B：功能平衡型

這套是在方案 A 基礎上加一些視覺層次與操作儀式感，但不讓人難用。

## 4.1 主選單

### 按鈕位置

- 中央是一個大圖騰圓盤
- 圓盤中心：`開始對戰`
- 圓盤左側：`編排小隊`
- 圓盤右側：`圖鑑`
- 圓盤下方：`養成 / 設定`
- 左上：玩家頭像與貨幣
- 右上：公告 / 活動 / 版本資訊

### User Flow

```text
開啟遊戲
  -> 進入圖騰主盤
    -> 點中央「開始對戰」
      -> 先進入備戰確認頁
        -> 確認隊長、家族、陣型
          -> 按右下「開始匹配」
```

### 和方案 A 的差別

- 開戰前多一個確認頁
- 適合這款遊戲的深度，不容易忘記自己帶錯隊

## 4.2 備戰與陣型編排

### 按鈕位置

- 中央：大型三層同心圓編隊盤
- 下方：目前已上陣成員列
- 左側分頁：`角色` `家族` `星級` `功能`
- 右側：詳細資訊與技能預覽
- 右下：`儲存`
- 右下上方：`推薦配置`
- 左上：`返回`

### User Flow

```text
進入備戰頁
  -> 看中央編隊盤
    -> 點下方已上陣成員
      -> 右側顯示角色詳情
        -> 左側切到家族分頁
          -> 替換一名成員
            -> 中央重新放置到內 / 中 / 外層
              -> 按「儲存」
                -> 按「開始匹配」
```

### 特色按鈕

- `推薦配置` 放在右側偏下
- 按下後不是直接亂排，而是給 2 到 3 套推薦：
  - 生存型
  - 平衡型
  - 爆發型

## 4.3 局內戰鬥 HUD

### 按鈕位置

- 左上：小地圖 + 縮圈計時
- 左側中段：任務 / 召喚條件追蹤
- 下方中央：能量條 + 隊長狀態
- 右下：主動技能大按鈕
- 右側中段：家族技能輪列
- 右上：剩餘隊伍數 / 擊殺數 / Boss 警報

### User Flow

```text
進入對局
  -> 探索地圖
    -> 左側看到召喚條件追蹤
      -> 決定要農怪、找工作台，還是去商店
        -> 遭遇戰鬥時看右側技能輪列
          -> 能量足夠時按右下主動技能
            -> 若達成 Boss 條件，右上出現警報
```

### 局內多一層花樣

- 家族技能不是單純圖示排排站
- 可做成一圈半弧狀，從右下往右上展開
- 這樣更像圍繞核心的世界觀

## 4.4 地圖互動介面

### 按鈕位置

- 互動面板從畫面下方滑出，不直接完全遮住戰場
- 左側：材料欄
- 中央：轉換或合成結果
- 右側：消耗、加成、世界加價資訊
- 右下：`確認處理`
- 左下：`離開`

### User Flow 範例：熔爐

```text
靠近熔爐
  -> 開啟下滑式面板
    -> 左側選材料
      -> 中央看到碎片轉換結果
        -> 右側顯示是否觸發地緣 +20%
          -> 按「確認處理」
            -> 關閉面板，立刻回戰場
```

### 這套的好處

- 玩家感覺自己沒有離開戰場太久
- 節奏比較順

## 4.5 對局結算介面

### 按鈕位置

- 中央：大名次
- 中央下方：本局成長摘要
- 左側：隊伍構成回顧
- 右側：死亡原因 / 傷害來源
- 左下：`回大廳`
- 右下：`再次出戰`

### User Flow

```text
對局結束
  -> 先看名次
    -> 往下看本局成長摘要
      -> 可切左側看最終陣型
      -> 可切右側看死亡原因
        -> 決定回大廳或再次出戰
```

## 5. 方案 C：圖騰儀式型

這套最有記憶點，但也最需要節制。

## 5.1 主選單

### 按鈕位置

- 整個主畫面是一個會微微旋轉的圖騰祭壇
- 中央核心：`開始儀式`
- 外圈四個方向按鈕：
  - 上：`隊長`
  - 右：`編隊`
  - 下：`圖鑑`
  - 左：`設定`
- 左下：版本資訊
- 右上：玩家資源與進度

### User Flow

```text
開啟遊戲
  -> 中央圖騰祭壇啟動
    -> 點中央核心「開始儀式」
      -> 展開四名隊長的選擇盤
        -> 選隊長
          -> 進入編隊確認
            -> 啟動匹配
```

### 特色

- 很有世界觀味道
- 第一次打開時很有「這遊戲有自己的臉」

## 5.2 備戰與陣型編排

### 按鈕位置

- 中央：大型三層旋轉圖騰盤
- 角色不是列表，而是圍繞外圍一圈的小肖像
- 左上：`返回`
- 右上：`隊伍總覽`
- 右下：`確認儀式`
- 左下：`清空重排`

### User Flow

```text
進入編隊儀式
  -> 外圍角色肖像環繞中央
    -> 點一名角色
      -> 角色肖像吸附到中央某層空槽
        -> 再選職責符號
          -> 盤面即時變色
            -> 完成後按「確認儀式」
```

### 按鈕位置細節

- 職責按鈕不放文字長條
- 改成三顆靠近盤面的符號鈕：
  - 盾牌 = 保護
  - 光環 = 支援
  - 尖刺 = 火力

## 5.3 局內戰鬥 HUD

### 按鈕位置

- 中央下方：核心能量圓環
- 右下：主動技能
- 右側：技能圖騰鏈
- 左上：小地圖
- 左下：消耗品輪盤
- 畫面中央外圈淡淡顯示小隊三層狀態

### User Flow

```text
進入對局
  -> 中央下方能量圓環持續提示節奏
    -> 玩家移動與輸出
      -> 當能量充滿或技能可用
        -> 右下主動技能亮起
          -> 左下輪盤可快速用藥
            -> 遇到重要互動點時，畫面中心浮出儀式選項
```

### 特色

- HUD 本身就像一個活著的圖騰
- 很符合這款遊戲的「環繞核心」概念

### 風險

- 太多圓形後，閱讀壓力會上升
- 需要明顯層級，不然玩家會眼花

## 5.4 地圖互動介面

### 按鈕位置

- 面板不是矩形，而是簡單的同心圓區塊式配置
- 中央：互動目標
- 左弧：投入材料
- 右弧：輸出結果
- 下方：`確認`
- 左上小字：`離開`

### User Flow 範例：雕像解鎖

```text
靠近雕像
  -> 開啟圖騰互動盤
    -> 中央顯示雕像角色
      -> 左弧放入需求材料
        -> 右弧預覽解鎖後效果
          -> 按下方「確認」
            -> 雕像破碎，角色加入圖鑑 / 可編隊
```

## 5.5 對局結算介面

### 按鈕位置

- 畫面中央是破碎或點亮的核心圖騰
- 中央文字：名次與評價
- 左側：本局路線回顧
- 右側：最關鍵三個決策節點
- 左下：`回到祭壇`
- 右下：`再次啟動`

### User Flow

```text
對局結束
  -> 顯示圖騰結果畫面
    -> 看名次與評價
      -> 左側回看自己本局進程
      -> 右側看敗因或勝因
        -> 決定回到祭壇或再次啟動
```

## 6. 三套方案比較

### 方案 A：極簡直覺型

- 優點：最快落地、最穩、最不容易做壞
- 缺點：個性最弱
- 最適合：先做 MVP、先驗證玩法

### 方案 B：功能平衡型

- 優點：實用與風格平衡最好
- 缺點：需要多花一點版面設計心力
- 最適合：第一個可玩的正式版本

### 方案 C：圖騰儀式型

- 優點：辨識度最高、最有作品味
- 缺點：最容易過度設計
- 最適合：在核心玩法已穩之後再上

---

## 7. 我建議的落地順序

如果你要我給最務實建議，我會這樣走：

1. **先做方案 A 的骨架**
先把主選單、編隊、HUD、商店、結算都放到能玩。

2. **再升級成方案 B**
把同心圓語言、家族資訊、召喚追蹤加進去。

3. **最後挑 1 到 2 個最有記憶點的區塊升成方案 C**
我建議優先升級：
- 主選單
- 編隊畫面

因為這兩個地方最能展現世界觀，而且不會像戰鬥 HUD 一樣影響操作壓力。
