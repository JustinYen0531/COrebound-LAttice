# COrebound LAttence 開發進度清單

本文件的用途，不是列出一堆零散待辦，而是把整個專案拆成幾個可以同步並行推進的共同階段。

這次的核心原則只有三個：

1. `你` 和 `Coding Agent` 不是分前後，而是同時進行
2. 每個階段不用做到 100 分，做到約 `70% 到 80%` 就可以進下一階段
3. 每個階段都要同時定義：
   - 你要做什麼
   - Coding Agent 要做什麼
   - 哪些小項目完成後，可以往下推

---

## 1. 這份清單怎麼用

你之後每次派任務時，只要先看三件事：

1. 我們現在在哪一個階段
2. 這個階段我和 Coding Agent 各自還差什麼
3. 哪些項目做到七八成，就可以讓下一階段先動起來

這樣的好處是：

- 不會卡在「一定要全部做完才敢往下」
- 可以讓規格、系統、素材一起長
- 比較符合你現在時間有限、要並行推進的狀況

---

## 2. 階段判定原則

每一個階段都用同一種判定方式：

- `完成核心骨架`
- `完成主要可見成果`
- `留下 20% 到 30% 的細修空間`

也就是說，進下一階段不代表上一階段完全結束，而是代表：

- 它已經足夠穩
- 足夠能被下一階段依賴
- 剩下的是微調，不是骨幹重做

---

## 3. 六個共同並行階段

## 階段一：方向定版與基礎骨架

### 本階段目標

- 把遊戲的主流程、主要介面、主要功能先定出穩定方向
- 同時把開發骨架與素材骨架都先搭起來

### 你要完成的部分

- 定掉主畫面有哪些主按鈕與子按鈕
- 定掉正式遊玩、管理介面、主畫面的三大關係
- 定掉 HUD 的主要構圖
- 定掉隊長頭像、生命值條、能量條的位置關係
- 定掉左滑技能欄、右滑物品欄的概念
- 定掉管理介面的資料夾結構
- 定掉圖鑑、世界故事、遊玩記錄的大分類
- 先確認第一批最優先素材要做哪些

### Coding Agent 要完成的部分

- 整理並統一現有規格文件
- 建立可持續擴充的文件結構
- 把 UI flow、管理 flow、互動 flow 收整成一致版本
- 把之後會接到的資料類型先盤點清楚
- 建立素材資料夾、說明文件、命名原則
- 預留之後系統與素材接線時需要的基本結構

### 本階段必須完成的小項目

- 主畫面 flow 明確
- 正式遊玩 flow 明確
- 管理介面 flow 明確
- 按鈕總表有第一版
- 資產資料夾有第一版
- 音樂與音效需求總表有第一版
- 文件之間不再互相矛盾

### 進入下一階段條件

- 三大頁面關係已經講得清楚
- 你和 Coding Agent 都知道下一步各自要做什麼
- 即使還有細節沒補滿，也已經不會推翻主方向

---

## 階段二：可玩主循環與可生成首批內容

### 本階段目標

- 一邊把最小可玩版本建立起來
- 一邊讓第一批視覺與聲音素材開始能被生成

### 你要完成的部分

- 開始準備第一批角色、怪物、世界主題素材
- 針對隊長、成員、怪物、Boss 確認第一版風格方向
- 先生成或篩選第一批主視覺
- 開始挑第一批主畫面 / HUD 的 UI 素材方向
- 開始處理主畫面主題曲、前期探索音樂、核心 UI 音效

### Coding Agent 要完成的部分

- 建立最小可玩的戰鬥主循環
- 完成進場、移動、碰撞、受傷、死亡、結算骨架
- 讓生命值、能量值、安全區等基礎系統跑起來
- 做出能承載 Placeholder 的主遊玩畫面
- 讓之後正式素材能替換進來的接口先存在

### 本階段必須完成的小項目

- 能從主流程進入一場戰局
- 一場戰局能開始、進行、結束
- Placeholder HUD 能顯示基本資訊
- 第一批角色與怪物素材有可選版本
- 第一批主題音樂與音效有可測版本

### 進入下一階段條件

- 遊戲已經不是只有文件，而是有基本可玩主循環
- 第一批素材不一定全定案，但方向已經足夠穩
- 主循環和素材方向彼此不衝突

---

## 階段三：HUD 操作成形與第一批素材接入

### 本階段目標

- 把戰鬥中的操作感做出來
- 同時讓第一批素材真正進入畫面，不再只是概念

### 你要完成的部分

- 補齊隊長頭像、角色頭像、HUD 零件的第一批正式圖
- 補齊主動技能、武器群組、週期技能的 icon 方向
- 確認生命值條、能量條、技能顯示的視覺風格
- 選定第一批真正要放進遊戲的圖與音效

### Coding Agent 要完成的部分

- 完成底部 HUD 核心構圖
- 完成頭像停留後的展圈邏輯
- 完成左滑技能欄與右滑物品欄
- 完成主動技能冷卻顯示
- 完成武器群組與週期技能的小圖示顯示
- 完成物品點按與拖曳使用邏輯
- 完成右側隊員狀態條的基本呈現
- 把第一批素材接進 HUD 和主遊玩畫面

### 本階段必須完成的小項目

- HUD 已能顯示並操作主要技能與物品
- 左滑與右滑互動不衝突
- 第一批 UI 素材已經進場
- 第一批音效已能覆蓋核心操作

### 進入下一階段條件

- 玩家已經能看懂並操作戰鬥主 HUD
- HUD 不再只是測試介面，而有第一版正式感
- 之後就算再細修，也不需要重做整套邏輯

---

## 階段四：管理介面、互動物件與第二批素材

### 本階段目標

- 把管理介面做成真正的中樞
- 同時把互動物件與第二批素材一起推進

### 你要完成的部分

- 製作或篩選管理介面的資料夾、分頁、圖鑑零件
- 製作互動物件主視覺：合成台、熔爐、雕像、商店、召喚裝置
- 製作第二批世界環境物件、材料圖示、地板材質
- 補齊中段需要的互動與管理音效
- 開始準備世界展示短片或技能展示短片

### Coding Agent 要完成的部分

- 完成管理介面開啟與關閉轉場
- 完成小隊頁的大圓盤與右側立繪布局
- 完成角色資訊、食譜、背包顯示
- 完成互動資料夾與其子頁入口
- 完成靠近互動物件的驚嘆號提示
- 完成從場上互動物件跳進管理頁指定子頁
- 完成圖鑑、世界故事、地圖縮影等入口
- 把第二批素材接入管理介面與場景互動點

### 本階段必須完成的小項目

- 管理介面可正常承載小隊、背包、互動、圖鑑
- 場上互動物件能正確導向各自頁面
- 第二批視覺素材已經開始替換場景與管理頁
- 互動音效已覆蓋主要操作

### 進入下一階段條件

- 管理介面已經可用，而不是只有框架
- 場上互動與管理頁已成功串接
- 第二批素材已足夠支撐世界感與系統感

---

## 階段五：主畫面外層體驗、紀錄系統與第三批素材

### 本階段目標

- 把整個遊戲外層體驗補完整
- 同時把第三批較高價值的展示素材推進

### 你要完成的部分

- 補齊主畫面正式視覺
- 補齊圖鑑與世界故事閱讀頁面的視覺素材
- 補齊宣傳圖、封面圖、商店圖
- 補齊新手導覽影片、訓練道場示範素材
- 補齊最終 Boss、勝利 / 失敗、成就相關音樂音效

### Coding Agent 要完成的部分

- 完成主畫面主按鈕與右側彈出子按鈕
- 完成 `開始遊玩`、`圖鑑`、`新手入門`、`遊玩記錄`、`設定`
- 完成 `Continue Game` 的存檔選擇
- 完成 `遊玩記錄` 與 `成就` 的呈現骨架
- 完成圖鑑與世界故事在主畫面端的入口
- 完成新手導覽與道場 / 沙盒入口
- 把第三批素材接入主畫面、記錄頁、教學頁、展示頁

### 本階段必須完成的小項目

- 主畫面可展開、收回、切進各子頁
- 遊玩記錄與成就有基本可見成果
- 圖鑑與世界故事能在非遊戲中閱讀
- 第三批高價值素材開始進場

### 進入下一階段條件

- 遊戲不再只有局內體驗，而有完整外層入口
- 教學、紀錄、圖鑑、主畫面都已經能用
- 對外展示需要的門面已經長出來

---

## 階段六：整合驗收、替換補齊與最終打磨

### 本階段目標

- 把前面五個階段長出來的系統與素材全部收束
- 做最後一輪整合、驗收、補洞與打磨

### 你要完成的部分

- 補完還缺的角色、怪物、環境、UI、宣傳素材
- 補完漏掉的音效、BGM、展示影片
- 針對風格不一致的素材做最後篩選或替換
- 針對外部展示、測試招募、商店頁所需資產做整理

### Coding Agent 要完成的部分

- 把所有批次素材完整掛載進遊戲
- 修正壞圖、壞音效、空頁、錯頁、錯連結
- 檢查主畫面、正式遊玩、管理介面三大流程
- 檢查圖鑑、世界故事、遊玩記錄、教學入口
- 檢查存檔、讀檔、成就、互動頁面資料
- 做基本效能、尺寸、排版與顯示穩定性修正

### 本階段必須完成的小項目

- 所有關鍵流程都能走通
- 所有關鍵頁面都有正式素材或可接受版本
- 主要聲音回饋已補齊
- 主要系統已不再依賴臨時 placeholder

### 完成判定

- 這個版本已可用於內部測試、展示、繼續細修
- 後續剩下的是優化與擴充，不是骨架重建

---

## 4. 你之後派工時最適合的問法

如果你之後要繼續一段一段推，我建議每次都用這種節奏：

1. 先說我們現在在 `第幾階段`
2. 指定這一輪要推哪一小塊
3. 分開看：
   - `你這邊要補什麼`
   - `Coding Agent 這邊要補什麼`
4. 最後只驗收一件事：
   - 這一小塊有沒有到 `70% 到 80%`，足夠讓下一塊開始

---

## 5. 這版清單和上一版最大的差異

這一版不再是：

- 前半段給 Coding Agent
- 後半段給你自己
- 做完一大塊再做下一大塊

這一版改成：

- 每個階段都同時有你的工作
- 每個階段都同時有 Coding Agent 的工作
- 每個階段都允許先做七八成，接著往下推

這樣才比較符合現在的現實狀況：

- 時間不多
- 事情很多
- 必須並行
- 不能等全部完美才動下一步
