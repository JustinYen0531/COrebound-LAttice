# COrebound LAttence 戰鬥 HUD 與操作骨架

本文件為戰鬥中 HUD 的正式定稿規格。

範圍嚴格限定:

- 底部隊長核心 HUD
- 三層同心圓展開
- 左滑 / 右滑互動
- 生命條 / 能量條
- 三類技能顯示邏輯
- 隊員狀態條
- 操作風險與防誤觸

本文件不處理:

- 主畫面
- 世界故事頁
- 管理介面內頁
- 商店 / 圖鑑 / 地圖

本文件只做文字策劃與 UI 互動邏輯定稿,不涉及程式碼。

---

## 0. 設計總綱

戰鬥 HUD 只有兩條最高原則:

1. **戰鬥中必須安靜。** 平常只能看到「核心、生命、能量」三件事。
2. **深度只在玩家主動要的時候才出現。** 滑鼠停留、滑動、靠近物件才展開,平常不存在。

違反這兩條的任何設計,直接退回。

---

## 1. HUD 構圖定稿

## 1.1 整體畫面分區

戰鬥畫面分為四個不重疊的功能區:

| 區域 | 位置 | 平常狀態 | 內容 |
| :--- | :--- | :--- | :--- |
| **核心 HUD** | 底部正中央 | 永久顯示 | 生命條 / 隊長頭像 / 能量條 |
| **展開同心圓** | 從核心 HUD 向上展開 | 隱藏,停留才展開 | 三層圈位 |
| **左滑抽屜** | 從核心向左展開 | 隱藏,左滑才展開 | 技能列 |
| **右滑抽屜** | 從核心向右展開 | 隱藏,右滑才展開 | 物品欄 + 隊員狀態條 |
| **資訊邊角** | 上方左右角 | 低調常駐 | 小地圖 / 毒圈 / 擊殺數 / 事件提示 |

四個區域的優先級:

- `核心 HUD` 永遠最優先,不可被任何抽屜或圓圈覆蓋
- `展開同心圓` 只能向上長,不可遮住核心 HUD 本體
- `左滑 / 右滑抽屜` 互斥,一次只能開一邊
- `資訊邊角` 永遠在背景層,不主動搶焦點

---

## 1.2 核心區尺寸與比例(設計基準)

以 1920 × 1080 為設計基準,其他解析度等比縮放。

### 核心三件組

底部正中央固定放置「生命條 ─ 隊長頭像 ─ 能量條」三者構成的整體。

```
┌─────────────────────────────────────────────────┐
│                                                 │
│                    [展開圓]                     │
│                    ↑ 向上長                     │
│                                                 │
├─────────────────────────────────────────────────┤
│   ┌──────┐         ╭───╮         ┌──────┐      │
│   │      │  ◄──── │隊長│ ────►  │      │      │
│   │ 生命 │         │頭像│         │ 能量 │      │
│   │  條  │         ╰───╯         │  條  │      │
│   └──────┘                       └──────┘      │
└─────────────────────────────────────────────────┘
```

### 尺寸基準

| 元件 | 基準尺寸 | 說明 |
| :--- | :--- | :--- |
| **隊長頭像** | `96 × 96` px | 圓形裁切,外環可顯示主動技能冷卻環 |
| **生命條** | 寬 `200`、高 `40` px | 緊貼頭像左側,與頭像垂直置中 |
| **能量條** | 寬 `200`、高 `40` px | 緊貼頭像右側,與頭像垂直置中 |
| **三者總寬** | 約 `624` px | 居中放置,左右各留約 `648` px 邊距 |
| **距底部** | `40` px | 核心區底緣距螢幕底部 |

### 錨點規則

- 整個核心三件組視為單一根節點
- 根節點水平置中、垂直貼底(留 `40` px)
- 生命條、能量條相對頭像「水平緊鄰、垂直置中」
- 不論展開與否,核心三件組位置不動

---

## 1.3 生命條規格

### 視覺

- 外框:深色細邊框(`2` px),帶輕微內陰影
- 填充:由左向右填充
- 顏色階梯(由滿到空):
  - `100% ~ 70%`:穩定主色(配合隊長代表色)
  - `69% ~ 30%`:警示橙
  - `29% ~ 0%`:危險紅 + 緩慢脈動
- 護盾量:在生命條外側額外顯示一條**灰色護盾覆蓋條**,長度等於當前護盾佔最大生命的比例

### 顯示資訊

- 平常只顯示**填充條**,不顯示數字
- 滑鼠停留在生命條上 `0.5s` 後,顯示「當前值 / 最大值」小字
- 護盾存在時,護盾條永久可見(不需停留)

### 護盾疊加規則

- 護盾覆蓋在生命條**外側上方**,不與生命填充重疊
- 護盾受擊優先扣減,扣完才扣生命
- 護盾條獨立用灰色,避免與生命色混淆

---

## 1.4 能量條規格

### 視覺

- 外框與生命條一致
- 填充:由左向右填充(與生命條同向,降低閱讀成本)
- 顏色:黃色「閃電」主色
- 能量不足施放主動技能時(低於該技能消耗),整條變暗 + 邊緣紅色提示

### 顯示資訊

- 平常只顯示填充條
- 滑鼠停留 `0.5s` 後,顯示「當前值 / 最大值」與「自然回復速率 / 秒」
- **能量臨界提示**:當能量足夠施放主動技能時,能量條右端亮起一個小亮點,表示「可用」

### 主動技能消耗預覽

- 當玩家按住 Space(預備施放)但能量不足時:
  - 能量條顯示一條「缺口標記」,標出還差多少
  - 主動技能圖示變暗

---

## 1.5 隊長頭像規格

### 平常狀態

- 圓形頭像,外環是主動技能冷卻環
- 頭像本身不旋轉、不脈動
- 隊長受致命傷(生命 < 30%)時,頭像外圍加紅色警示環

### 冷卻環

- 頭像外環以順時針掃過的方式表示主動技能冷卻進度
- 冷卻中:環為暗色 + 掃過的已冷卻部分亮起
- 可施放:整環亮起 + 微微脈動
- 能量不足:環維持可施放色,但頭像整體變暗

### 點擊行為

- 點擊頭像 = **施放隊長主動位移技能**(等於 Space 鍵)
- 這是滑鼠玩家的主要施放路徑

## 2. 三層同心圓展開系統

## 2.1 展開的觸發與時序

展開完全由「滑鼠停留在核心區」驅動,不靠點擊。

### 觸發區域

- 滑鼠停留在「隊長頭像 + 生命條 + 能量條」構成的核心矩形區域內
- 停留區域包含三者及其間隙,但不包含外圍邊距

### 展開時序(固定)

| 階段 | 觸發條件 | 動作 | 時長 |
| :--- | :--- | :--- | :--- |
| **階段 0:提示** | 滑鼠進入核心區 | 顯示「可左滑 / 可右滑」細微提示 | 立即 |
| **階段 1:最內圈** | 滑鼠停留 `0.5s` | 最內圈從頭像上方淡入展開 | `0.25s` |
| **階段 2:中間圈** | 最內圈展開完成後 `0.2s` | 中間圈淡入展開 | `0.25s` |
| **階段 3:最外圈** | 中間圈展開完成後 `0.2s` | 最外圈淡入展開 | `0.25s` |

總展開時間:`0.5 + 0.25 + 0.2 + 0.25 + 0.2 + 0.25 = 1.65s`(由停留到全開)

### 收回時序

- 滑鼠離開核心區 + 離開已展開的圓圈區域
- 等待 `0.3s` 緩衝(避免邊緣抖動)
- 由外向內依序收回,每層 `0.15s`

### 中斷規則

- 展開過程中,若玩家**移動小隊**(WASD)或**施放主動技能**(Space),展開立即中止並收回
- 展開過程中,若滑鼠離開核心區,展開中止並收回
- 已全開狀態下移動小隊,圓圈保持(不移動時戰術閱讀需要),但**滑動抽屜不得開啟**

---

## 2.2 三層圓圈的視覺結構

每一圈都是一個獨立的同心圓環,對應小隊的層位。

### 層位對應

| 圈層 | 戰術意義 | 半徑(基準) |
| :--- | :--- | :--- |
| **最內圈** | 內層成員(生存力強,難輸出) | 頭像外 `+48` px |
| **中間圈** | 中層成員(平衡) | 最內圈外 `+40` px |
| **最外圈** | 外層成員(最先受擊,最易輸出) | 中間圈外 `+40` px |

### 每圈的組成

每一圈分為三個顏色槽位,對應三種職責:

- 🔵 **藍色槽:保護位**
- 🔴 **紅色槽:火力位**
- 🟡 **黃色槽:補給位**(即支援/資源位)

> 註:本文件採用補充規格文件的最新定義(藍/紅/黃),與 GDD 初版的綠/黃/紅不同。後續所有 HUD 與管理介面統一用藍/紅/黃。

### 槽位的閱讀方式

玩家看到每一圈時,應能直覺讀出:

- 這圈有沒有藍位成員(能否扛傷)
- 這圈火力重不重(紅位多寡)
- 這圈補給是否完整(黃位多寡)

槽位不是裝飾,是**戰術情報**。

### 槽位填充規則

- 有成員的槽位:對應顏色亮起 + 顯示成員小頭像(或代表紋樣)
- 無成員的槽位:暗色空槽
- 該層該職責的成員**死亡**:槽位變為虛線外框(保留位置記憶,提示曾有人)
- 該層該職責的成員**護盾存在**:槽位外框轉為灰色

### 槽位排序

- 同一圈內,槽位順序固定為 `藍 → 紅 → 黃`(順時針)
- 不依成員動態重排,避免戰鬥中閃爍

---

## 2.3 旋轉規則

每一圈都會緩慢旋轉,傳達「活性」而非混亂。

### 旋轉參數(基準)

| 圈層 | 方向 | 速度 | 視覺感受 |
| :--- | :--- | :--- | :--- |
| 最內圈 | 順時針 | 最慢 | 核心、凝聚、穩定 |
| 中間圈 | 逆時針 | 中速 | 平衡 |
| 最外圈 | 順時針 | 最快 | 活躍、鬆動 |

### 旋轉原則

- 相鄰圈方向相反,避免視覺同步導致呆板
- 越外層越快,建立「越外圈越高速,越內圈越凝聚」的視覺語言
- 速度必須**偏慢且穩定**,不可快到像陀螺失控
- 旋轉是裝飾層,**不可影響槽位判讀**(槽位內容不隨旋轉模糊)

### 戰鬥中旋轉調整

- 小隊移動中:旋轉速度 `+30%`(傳達推進感)
- 小隊靜止:旋轉速度正常
- 隊長主動技能施放中:全部圈層暫停旋轉 `0.5s`(視覺強調)

---

## 2.4 展開後的點擊行為

當三層圓圈已展開(任意圈數)時:

- 點擊圓圈**任一位置**(不需精準點某個槽位) → **開啟管理介面**
- 點擊特定槽位 → 開啟管理介面並**直接跳到該成員詳情頁**

### 點擊容差

- 點擊判定放寬到整個圓盤區,不要求精準命中槽位
- 目的:戰鬥中手指/滑鼠不穩也不會點空

### 點擊 vs 施放

- 點擊頭像本體 = 施放主動技能
- 點擊頭像外的圓圈區 = 進管理介面
- 兩者區域不重疊,不會誤觸

## 3. 滑動與停留互動規則

## 3.1 三種互動模式

核心區支援三種獨立的玩家意圖:

| 模式 | 觸發 | 用途 |
| :--- | :--- | :--- |
| **停留** | 滑鼠在核心區不動 | 展開同心圓(閱讀陣型) |
| **左滑** | 滑鼠由核心區向左拖曳 | 展開技能列(管理能量消耗) |
| **右滑** | 滑鼠由核心區向右拖曳 | 展開物品欄 + 隊員狀態條(補給) |

---

## 3.2 停留判定

### 提示階段

- 滑鼠進入核心區 → 立即顯示「可左滑 / 可右滑」細微提示
- 提示樣式:核心區左右兩側各一個小箭頭 + 文字,半透明
- 提示**不巨大**,目的是告知「這裡可以滑」,不是強制導引

### 展開階段

- 滑鼠在核心區**連續不動**滿 `0.5s` → 進入展開時序(見 §2.1)
- 「連續不動」的容差:滑鼠位移 < `8` px 視為不動

### 提示與展開的關係

- 提示出現後,若玩家**立刻滑動**,則提示消失、進入滑動模式,不展開圓圈
- 提示出現後,若玩家**繼續不動**,則 `0.5s` 後開始展開圓圈,提示同時消失

---

## 3.3 滑動判定

### 滑動門檻

- 從核心區起始,向左或向右拖曳超過 `40` px → 認定為滑動意圖
- 拖曳 `8 ~ 40` px 之間 → 不觸發任何模式(防誤觸緩衝帶)
- 垂直拖曳(上下) → 不觸發左右滑(留給未來其他用途)

### 滑動方向鎖定

- 一旦認定為左滑或右滑,本次拖曳**鎖定該方向**
- 玩家中途反向,不會切換到另一邊抽屜
- 必須鬆開滑鼠、重新從核心區起始,才能換邊

### 滑動展開的動畫

- 抽屜跟隨滑鼠拖曳量**即時展開**(1:1 追隨),直到全開
- 鬆開滑鼠時:
  - 若展開量 > `50%` → 自動補滿到全開
  - 若展開量 < `50%` → 自動收回
- 全開後,抽屜**保持開啟**,直到玩家主動收回

---

## 3.4 互斥原則(關鍵)

**左滑與右滑一次只能開一邊。**

### 規則

- 左滑抽屜開啟時,右滑抽屜**不能**開啟(滑向右邊無反應)
- 右滑抽屜開啟時,左滑抽屜**不能**開啟
- 展開同心圓(停留)與滑動抽屜**可同時存在**,但:
  - 滑動抽屜開啟時,同心圓不再繼續展開新的圈層
  - 已展開的圈層保持,但點擊圓圈進管理介面的行為**暫停**(避免與抽屜操作衝突)

### 切換方式

- 要從左滑切到右滑,必須**先收回左滑抽屜**
- 收回方式:再次點擊核心區、或按 Esc、或拖曳回核心區

### 為什麼互斥

- 畫面不會爆開
- 玩家不會同時被兩種操作模式拉扯
- 操作意圖始終單一、清楚

---

## 3.5 抽屜收回條件

抽屜會在以下情況自動收回:

| 條件 | 行為 |
| :--- | :--- |
| 玩家再次點擊核心區 | 立即收回 |
| 按下 `Esc` | 立即收回 |
| 小隊開始移動(WASD) | `0.5s` 後收回(給玩家收尾時間) |
| 進入戰鬥碰撞(受擊) | 不自動收回(玩家可能正在喝藥) |
| 開啟管理介面 | 收回所有抽屜與圓圈 |
| 滑鼠離開整個 HUD 區超過 `3s` | 收回(防掛機殘留) |

---

## 3.6 互動狀態機(總覽)

HUD 互動的有限狀態:

```
                    [IDLE 平常]
                         │
            ┌────────────┼────────────┐
            ▼            ▼            ▼
      滑鼠進入核心     左滑>40px     右滑>40px
            │            │            │
            ▼            ▼            ▼
      [HOVER 提示]  [LEFT_OPEN]  [RIGHT_OPEN]
            │            │            │
     停留0.5s      ┌─────┴─────┐  ┌─────┴─────┐
            ▼      ▼           ▼  ▼           ▼
     [RING 展開中] 操作技能   收回    操作物品   收回
            │
     全展開完成
            │
            ▼
      [RING 全開]
            │
     點擊圓圈
            │
            ▼
      [開啟管理介面]
```

**互斥規則在狀態機的體現**:

- `LEFT_OPEN` 與 `RIGHT_OPEN` 之間沒有直接轉移邊
- `RING` 展開中若收到滑動輸入 → 中止展開、轉移到對應抽屜狀態

## 4. 三類技能顯示邏輯

## 4.1 三類技能的分層

| 層級 | 類型 | 顯示方式 | 平常可見性 |
| :--- | :--- | :--- | :--- |
| **第一層** | 隊長主動位移技能 | 正式按鈕(頭像冷卻環) | **永久可見** |
| **第二層** | 武器群組技能 | 能量條附近小圖示 | **永久可見但小** |
| **第三層** | 週期技能 / 自動施法 | 能量條附近小圖示(累積環) | **永久可見但小** |

三層在視覺權重上必須**明顯拉開差距**,讓玩家一眼分得出「我要按的」vs「系統在跑的」。

---

## 4.2 第一層:隊長主動位移技能

### 顯示位置

- **頭像本身就是它的按鈕**
- 頭像外環 = 冷卻環
- 不另開獨立技能圖示,避免與頭像搶空間

### 狀態顯示

| 狀態 | 頭像外環 | 頭像本體 |
| :--- | :--- | :--- |
| 可施放(冷卻好 + 能量足) | 整環亮起 + 微脈動 | 正常 |
| 冷卻中 | 暗環 + 順時針掃過已冷卻部分 | 正常 |
| 能量不足 | 環維持可施放色 | 整體變暗 |
| 施放延遲中(防連發 0.5s) | 環快速閃一下 | 微暗 |

### 操作

- 點擊頭像 / 按 Space → 施放
- 按住 Space(預備)但能量不足 → 頭像變暗 + 能量條顯示缺口標記

### 設計意圖

這是戰鬥中**唯一**像「主技能按鈕」的存在,尺寸最大、最完整,玩家隨時知道它能不能用。

---

## 4.3 第二層:武器群組技能

### 顯示位置

- 緊貼能量條**上方**,水平排列一排小圖示
- 每個武器家族一個圖示(最多 4 個,因隊伍配置而異)
- 圖示尺寸:約 `36 × 36` px,明顯小於頭像

### 圖示內容

- 家族紋樣(護盾 / 多發 / 直線 / 地雷 / 激光)
- 圖示左下角小數字:該家族目前啟用的星級(1/2/3★)
- 圖示外圈:冷卻進度環(轉圈表示)

### 狀態顯示

| 狀態 | 圖示 | 外圈 |
| :--- | :--- | :--- |
| 啟用中 | 亮起 | 冷卻環轉動 |
| 玩家手動關閉 | 變灰 | 靜止 |
| 家族人數不足(被強制停用) | 變灰 + 鎖頭小圖 | 靜止 |
| 持續耗能中 | 亮起 + 小幅流動光 | 流動效果 |

### 互動(在左滑抽屜中)

- 平常小圖示**只能看,不能關**
- 左滑展開後,小圖示放大、可點擊**單獨關閉**某個武器群組
- 關閉後該家族子彈停止發射(節省能量)

### 設計意圖

武器群組是副技能,平常不搶畫面,但玩家要能隨時瞄一眼知道「哪些在跑、冷卻到哪」。

---

## 4.4 第三層:週期技能 / 自動施法

### 顯示位置

- 緊貼能量條**下方**,水平排列一排小圖示
- 與武器群組上下分開,避免混淆
- 圖示尺寸:約 `32 × 32` px,略小於武器群組

### 圖示內容

- 週期技能 / 自動施法的代表紋樣
- 圖示外圈:**累積充能環**(非冷卻環)

### 狀態顯示(週期技能)

週期技能靠**移動距離**充能,重點是「累積到哪」,不是「冷卻到哪」。

| 狀態 | 圖示 | 充能環 |
| :--- | :--- | :--- |
| 充能中 | 亮起 | 環由空逐漸填滿 |
| 充能接近完成(>80%) | 提高 brightness | 環幾乎全滿 |
| 充能滿(待觸發) | 脈動 | 全滿 + 脈動 |
| 已觸發釋放中 | 全亮 | 短暫全滿後歸零,重新累積 |

### 狀態顯示(自動施法)

自動施法靠**傷害 Tick 累積**觸發,顯示類似但語意不同。

| 狀態 | 圖示 | 累積環 |
| :--- | :--- | :--- |
| 累積 Tick 中 | 亮起 | 環依 Tick 進度填充 |
| 環滿即將觸發 | 脈動 | 全滿 |
| 觸發冷卻中 | 變暗 | 靜止 |

### 設計意圖

這類技能**不需要按鈕感**,玩家只需知道「快觸發了 / 已觸發」。用累積環 + 脈動傳達節奏,不用數字。

---

## 4.5 三類技能的視覺權重對照

| 維度 | 主動技能 | 武器群組 | 週期/自動 |
| :--- | :---: | :---: | :---: |
| 尺寸 | `96px` | `36px` | `32px` |
| 對比 | 最強 | 中 | 較弱 |
| 可點擊 | ✅(施放) | ⬜(左滑後可關) | ❌ |
| 動效 | 冷卻掃描 | 冷卻轉圈 | 充能脈動 |
| 戰鬥中搶眼度 | 高 | 低 | 最低 |

這個對比必須夠大,否則玩家分不清「我該按哪個」。

## 5. 隊員狀態條(右滑抽屜專屬)

## 5.1 出現時機

隊員狀態條**只在右滑抽屜開啟時出現**,平常不存在。

### 位置

- 右滑抽屜的右側
- 與左側物品欄構成「左取右用」的補給佈局

---

## 5.2 顯示數量

- 平常先顯示 `4 ~ 5` 名隊員
- 超過的隊員用**捲動/卷軸**切換
- 不一次攤開全部 8 名,避免視覺過載

### 捲動規則

- 滾輪 / 拖曳卷軸切換
- 目前「正在受傷」或「護盾存在」的隊員**自動置頂**,優先顯示
- 受致命傷(<30% 生命)的隊員額外紅色高亮

---

## 5.3 每位隊員的顯示內容

每位隊員狀態條至少包含:

| 元件 | 顯示 |
| :--- | :--- |
| **小頭像** | 隊員肖像 |
| **生命條(迷你)** | 當前生命比例 |
| **職責色標** | 藍/紅/黃小圓點(對應所屬槽位) |
| **層位標記** | 內/中/外 小標籤 |
| **異常狀態** | 圖示(沉默/眩暈/減速/燃燒等) |
| **可否使用物品** | 灰階表示無法指定(如已死亡) |

### 不顯示的內容

- 不顯示詳細數值(戰鬥中不需要)
- 不顯示技能列表(那是管理介面的事)
- 不顯示食譜 / 升星資訊

---

## 5.4 互動:拖曳使用物品

右滑抽屜的核心操作是**拖曳補給**。

### 操作流程

```
左側物品欄(藥水圖示)
   │
   │ 玩家按住拖曳
   ▼
右側某位隊員狀態條
   │
   │ 放下
   ▼
該隊員生效(喝藥)
```

### 拖曳中的視覺

- 被拖曳的藥水圖示跟隨游標
- 經過的可指定隊員**高亮**(表示可放下)
- 不可指定的隊員(死亡)維持灰階、不高亮
- 放下成功:短暫綠光確認
- 放下失敗(放空):藥水回到物品欄

### 快捷使用(不拖曳)

- 點擊物品欄的藥水 → **直接對隊長使用**(預設目標)
- 拖曳到隊員 → **對該隊員使用**
- 雙擊藥水 → **對生命最低的隊員使用**(智能補給)

---

## 6. 操作風險與防誤觸規則

## 6.1 戰鬥 HUD 的操作風險地圖

| 風險 | 場景 | 嚴重度 |
| :--- | :--- | :---: |
| **誤開抽屜** | 激烈戰鬥中滑鼠晃動意外觸發左/右滑 | 高 |
| **誤展圓圈** | 滑鼠短暫停在核心區,圓圈彈出擋視線 | 中 |
| **誤進管理介面** | 想點技能卻點到展開的圓圈 | 高 |
| **誤喝藥** | 拖曳藥水放錯隊員 | 中 |
| **抽屜卡住** | 戰鬥中抽屜沒自動收回,擋住視線 | 高 |
| **頭像難點** | 頭像小,想施放技能卻點空 | 中 |

以下規則逐一對應這些風險。

---

## 6.2 防誤觸:緩衝帶與門檻

### 滑動緩衝帶

- 拖曳 `0 ~ 8` px:**完全不反應**(噪聲過濾)
- 拖曳 `8 ~ 40` px:**僅預覽不鎖定**(抽屜微微跟手但未觸發)
- 拖曳 `> 40` px:**鎖定為滑動意圖**

### 停留容差

- 滑鼠位移 `< 8` px 視為「不動」
- 避免手抖導致停留判定失敗

### 點擊 vs 拖曳

- 滑鼠按下到放開:
  - 位移 `< 8` px → 認定為**點擊**
  - 位移 `> 8` px → 認定為**拖曳**(可能觸發滑動)
- 點擊頭像 = 施放技能;拖曳頭像 ≠ 施放

---

## 6.3 防誤展圓圈

### 戰鬥碰撞中抑制展開

- 小隊**正在受擊**(最近 1 秒內有碰撞傷害)時:
  - 圓圈**不展開**,即使滑鼠停留
  - 顯示提示「戰鬥中,展開已暫停」(小字、不搶眼)
- 目的:激戰中不讓圓圈彈出擋視線

### 移動中抑制展開

- 小隊**正在移動**(WASD 輸入中)時:
  - 圓圈**不展開**
  - 已展開的圈層**保持**(玩家邊走邊看陣型是合理的)
  - 但不展開**新的**圈層

### 施放技能中抑制

- 隊長主動技能施放動作進行中 → 圓圈不展開、已展開的暫停旋轉

---

## 6.4 防誤進管理介面

### 點擊區分離

- **頭像本體**的點擊 = 施放技能(永遠)
- **圓圈區域**的點擊 = 進管理介面(僅在全開時)
- 兩者區域**不重疊**:頭像是最內的小圓,圓圈在頭像外側

### 展開未完成時不觸發

- 只有當三層**全部展開完成**後,點擊圓圈才會進管理介面
- 展開過程中點擊 → 視為「取消展開」,收回圓圈
- 避免玩家想看陣型卻意外跳進管理介面

### 抽屜開啟時暫停

- 左滑或右滑抽屜開啟時,點擊圓圈**不進管理介面**
- 必須先收回抽屜,才能透過圓圈進管理介面
- 避免操作意圖混亂

---

## 6.5 防誤喝藥

### 拖曳目標高亮

- 拖曳藥水時,只有**可生效**的隊員高亮
- 死亡隊員、無法受效的隊員維持灰階,放上去**無反應**
- 放下失敗(放空處):藥水回到物品欄,**不消耗**

### 確認機制(大藥水)

- **大生命藥水 / 大能量藥水 / 大混合藥水**(昂貴)使用時:
  - 拖曳放下後,**不自動生效**
  - 該隊員狀態條出現「確認使用?」小按鈕,`3s` 內點擊確認才生效
  - `3s` 過期或按 Esc → 取消,藥水退回
- 小藥水無需確認(便宜,重操作流暢度)

### 連續使用防呆

- 同一隊員生命已滿時,對其拖曳生命藥水:
  - 該隊員**不高亮**(提示無效)
  - 硬放上去 → 無反應,藥水退回
- 避免浪費昂貴藥水

---

## 6.6 防抽屜卡住

### 自動收回觸發(重申 §3.5)

抽屜在以下情況自動收回,避免戰鬥中卡視線:

- 小隊開始移動 → `0.5s` 後收回
- 滑鼠離開 HUD 區超過 `3s` → 收回
- 開啟管理介面 → 收回
- 按 Esc → 立即收回

### 戰鬥碰撞中不收回

- 注意:受擊時**不**自動收回(玩家可能正在喝藥自救)
- 這是刻意的,讓玩家在挨打時能完成補給

### 抽屜不遮核心

- 左滑抽屜最右側、右滑抽屜最左側,**都不得覆蓋核心三件組**
- 抽屜從核心兩側向外展開,核心永遠可見
- 確保玩家喝藥時還能看見生命 / 能量

---

## 6.7 防頭像難點

### 點擊容差擴大

- 頭像點擊判定區域 = 頭像本體 `+ 8` px 外擴
- 戰鬥中手抖也能點到

### 鍵鼠雙路徑

- 點擊頭像 = 施放
- Space 鍵 = 施放
- 兩條路徑等價,玩家可選慣用手

### 點空反饋

- 點擊核心區但未命中頭像(命中生命條/能量條/間隙):
  - 無副作用
  - 不顯示錯誤提示(戰鬥中不彈錯誤訊息)

---

## 6.8 視覺干擾防護

### 展開圓圈的透明度

- 展開的同心圓**半透明**(約 `80%` 不透明度)
- 不完全遮擋背後的戰場
- 玩家仍能看到敵方小隊接近

### 抽屜的背景

- 抽屜背景使用**深色半透明**面板
- 邊緣有輕微發光,與戰場區隔
- 但不使用全黑遮罩(那是管理介面的事)

### 資訊邊角分層

- 小地圖、毒圈、擊殺數等邊角資訊
- **永遠在背景層**,HUD 抽屜與圓圈展開時不消失
- 但**自動降低亮度**(`-40%`),避免搶焦

---

## 7.2 停留 / 滑動後出現

```
戰鬥進行中
  │
  ├─ 平常狀態

  │         │
  │         ├─ 繼續停留 0.5s
  │         │    └─ 同心圓由內而外展開
  │         │         └─ 三圈顯示藍/紅/黃槽位
  │         │              └─ 各圈不同方向旋轉
  │         │                   └─ 點擊圓圈任意處
  │         │                        └─ 開啟管理介面(小隊頁)
  │         │
  │         ├─ 向左拖曳 > 40px
  │         │    └─ 左滑抽屜展開(技能列)
  │         │         └─ 武器群組圖示放大
  │         │              └─ 可點擊單獨關閉武器
  │         │
  │         └─ 向右拖曳 > 40px
  │              └─ 右滑抽屜展開(物品欄 + 隊員狀態條)
  │                   └─ 拖曳藥水到隊員
  │                        └─ 該隊員喝藥生效
  │
  ├─ 靠近互動物件
  │    └─ 主畫面出現驚嘆號
  │         └─ 點擊驚嘆號
  │              └─ 開啟管理介面(互動分頁)
  │
  └─ 任意時刻按 Esc
       └─ 收回所有抽屜與圓圈,回到平常狀態
```
