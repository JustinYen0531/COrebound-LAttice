# COrebound LAttence 管理介面與互動資料夾骨架

本文件只處理 `管理介面` 本身的分頁系統與互動導頁規則。

這一版刻意不碰戰鬥 HUD，不處理局內即時技能顯示，而是專心把管理頁的骨架釘穩。

---

## 1. 文件目標

這份文件要交付四件事：

1. `管理介面資訊架構`
2. `各資料夾層級`
3. `合成台 / 熔爐 / 雕像 / 商店 / 召喚的入口規則`
4. `接近互動物件時怎麼跳頁`

這代表它不是美術稿，也不是程式規格，而是後續細化畫面、流程稿、互動稿時的上位骨架。

---

## 2. 管理介面的定位

管理介面是正式遊玩中的深度系統頁。

它的責任是承接：

- `小隊`
- `背包`
- `互動`
- `圖鑑`
- `世界故事`
- `地圖縮影`

它不是另一套獨立 UI，而是從玩家的核心隊伍系統延伸出來的資料夾式管理頁。

---

## 3. 管理介面資訊架構

## 3.1 第一層主資料夾

管理介面的第一層主資料夾固定為 5 個：

- `小隊`
- `背包`
- `互動`
- `圖鑑`
- `地圖`

這 5 個主資料夾就是整個管理介面的主骨架。

### 第一層的定位

- `小隊`
  - 負責角色、位置、配置、食譜與培養資訊

- `背包`
  - 負責材料、消耗品、任務物與追蹤資源

- `互動`
  - 負責所有場景設施與世界互動入口

- `圖鑑`
  - 負責靜態知識、設定資料與世界故事閱讀

- `地圖`
  - 負責當前局勢的縮影、互動點與危險區

---

## 3.2 版面分工

管理介面不是所有頁都用同一套構圖。

### 小隊頁專屬構圖

只有 `小隊` 頁使用：

- 左邊：大型圓盤
- 右邊：角色立繪與詳細資訊

這套構圖專門用來說明小隊核心結構，不應套用到其他主資料夾。

### 非小隊頁構圖

以下頁面都改用資料夾式構圖：

- `背包`
- `互動`
- `圖鑑`
- `地圖`

其閱讀分工為：

- 左邊：第一層主資料夾書籤
- 中央：第二層分頁與主要內容
- 右邊：補充資訊、說明、預覽或操作結果

---

## 3.3 管理介面的閱讀順序

第一版建議固定為：

```text
左側第一層主資料夾
  -> 中央第二層子分類
    -> 中央主內容
      -> 右側補充資訊 / 操作結果 / 詳細說明
```

這個順序的好處是：

- 先決定自己在管理什麼
- 再決定看哪個子類
- 最後進入細節內容

---

## 4. 各資料夾層級

## 4.1 小隊

```text
小隊
  -> 總覽
  -> 成員
  -> 配置
  -> 食譜
```

### 子頁職責

- `總覽`
  - 顯示大型圓盤、小隊圈層與目前編隊全貌

- `成員`
  - 查看各成員立繪、定位、狀態與細節

- `配置`
  - 調整成員所在圈層與藍 / 紅 / 黃槽位

- `食譜`
  - 直接查詢角色合成與培養需求

---

## 4.2 背包

```text
背包
  -> 材料
  -> 消耗品
  -> 任務物
  -> 追蹤中
```

### 子頁職責

- `材料`
  - 一般合成、升級、解鎖素材

- `消耗品`
  - 藥水、戰鬥可使用物件、一次性補給

- `任務物`
  - 特殊條件、世界事件、階段性進程道具

- `追蹤中`
  - 玩家主動標記正在缺少或近期想補齊的資源

---

## 4.3 互動

```text
互動
  -> 合成
  -> 熔爐
  -> 雕像
  -> 商店
  -> 召喚
```

### 子頁職責

- `合成`
  - 角色、素材或配方相關的製作入口

- `熔爐`
  - 材料熔煉、轉換或重構入口

- `雕像`
  - 解鎖、供奉、增益或世界機關啟動入口

- `商店`
  - 購買藥水、出售材料、交換資源入口

- `召喚`
  - 召喚物、事件、Boss 條件或特殊裝置入口

---

## 4.4 圖鑑

```text
圖鑑
  -> 成員圖鑑
  -> 怪物圖鑑
  -> 世界圖鑑
  -> 材料圖鑑
  -> 機制圖鑑
  -> Boss圖鑑
  -> 世界故事
```

### 子頁職責

- `成員圖鑑`
  - 查小隊成員、來源、定位、設計資料

- `怪物圖鑑`
  - 查怪物分類、特性與世界歸屬

- `世界圖鑑`
  - 查四個世界的構成、視覺與生態

- `材料圖鑑`
  - 查材料來源、用途、對應配方

- `機制圖鑑`
  - 查戰鬥規則、狀態、系統說明

- `Boss圖鑑`
  - 查守護者與進階敵人資訊

- `世界故事`
  - 收斂輪迴敘事、世界背景與多次重啟後解鎖的文本

---

## 4.5 地圖

```text
地圖
  -> 縮影
  -> 互動點
  -> 危險區
  -> 事件區
```

### 子頁職責

- `縮影`
  - 顯示當前局勢的簡化戰場地圖

- `互動點`
  - 顯示合成台、熔爐、雕像、商店、召喚裝置等位置

- `危險區`
  - 顯示高風險區域、地形威脅與壓力帶

- `事件區`
  - 顯示目前可觸發或已活化的局內事件

---

## 4.6 深度原則

管理介面的深度先控制在兩層：

- 第一層：主資料夾
- 第二層：子分類

暫時不要做第三層、第四層連續巢狀。

如果內容很多，優先用：

- 分頁
- 篩選
- 搜尋

而不是把資料夾越鑽越深。

## 5. 互動設施的入口規則

## 5.1 共通規則

所有場景設施都走同一套規則：

1. 玩家平常就能在 `互動` 資料夾看到該入口
2. 未接近時入口不消失，但標記為不可操作
3. 接近對應設施時，該入口亮起或轉為可操作
4. 點場上的互動提示，直接打開管理介面並跳到指定子頁

這樣可以維持結構穩定，玩家也不會覺得功能忽隱忽現。

---

## 5.2 未接近時的狀態

玩家沒有靠近對應設施時，`互動` 資料夾仍應顯示：

- `目前未接近對應設施`
- `暫不可操作`
- `請靠近對應設施後啟用`

### 文案語氣原則

- 不像報錯
- 不像功能消失
- 比較像系統存在，但尚未連接到現場

若玩家硬要點進去，可顯示：

- `暫不開放`
- `尚未連接到對應設施`

---

## 5.3 接近時的狀態

玩家接近設施後，該設施需要同步出現兩種變化：

1. `場上提示出現`
2. `互動資料夾對應子頁亮起`

這樣玩家不管是從場上進，還是先打開管理介面看，都能理解自己目前能操作什麼。

---

## 6. 接近互動物件時的跳頁規則

## 6.1 場上提示

當玩家進入互動範圍時，主畫面先出現一個類似 `驚嘆號` 的可感知提示。

這個提示的意義是：

- 附近有可互動設施
- 點擊後可快速進入對應管理頁

它不是直接把操作塞在主畫面上，而是引導玩家進入既有管理系統。

---

## 6.2 跳頁流程

```text
玩家接近互動物件
  -> 主畫面出現驚嘆號提示
    -> 玩家點擊提示
      -> 打開管理介面
        -> 自動切到第一層「互動」
          -> 自動切到對應第二層子頁
            -> 將該設施內容放在中央主區
            -> 在右側顯示補充說明 / 操作結果 / 條件資訊
```

---

## 6.3 對應表

```text
接近合成台 -> 互動 / 合成
接近熔爐   -> 互動 / 熔爐
接近雕像   -> 互動 / 雕像
接近商店   -> 互動 / 商店
接近召喚裝置 -> 互動 / 召喚
```

---

## 6.4 進入後的焦點規則

跳進指定子頁後，介面焦點應放在三件事：

1. `該設施現在能做什麼`
2. `玩家是否具備操作條件`
3. `執行後會產生什麼結果`

因此建議：

- 中央主區顯示主要操作內容
- 右側欄顯示材料、條件、結果、提示

這樣跳頁不是只有「打開到對的地方」，而是真的立即可理解。

---

## 6.5 返回規則

從互動子頁離開時，應允許兩種返回方式：

- `回到互動資料夾首頁`
- `回到上一個管理頁上下文`

如果這次是從場上互動提示直接進來，也可以額外提供：

- `返回戰場`

這樣玩家不會覺得自己被困在一個突然打開的設施頁裡。

---

## 7. 世界故事入口的定位

`世界故事` 放在 `圖鑑` 之下，而不是 `互動` 或 `地圖`。

原因很單純：

- 它是靜態閱讀與收斂內容
- 它對戰鬥不提供即時操作
- 它承接的是輪迴敘事與世界觀解鎖

因此定位為：

```text
圖鑑 -> 世界故事
```

後續如果要把每次重啟的故事片段逐步解鎖，也應優先沿用這個入口。

## 8. 第一版總摘要

```text
管理介面
  -> 小隊
    -> 總覽 / 成員 / 配置 / 食譜
  -> 背包
    -> 材料 / 消耗品 / 任務物 / 追蹤中
  -> 互動
    -> 合成 / 熔爐 / 雕像 / 商店 / 召喚
  -> 圖鑑
    -> 成員圖鑑 / 怪物圖鑑 / 世界圖鑑 / 材料圖鑑 / 機制圖鑑 / Boss圖鑑 / 世界故事
  -> 地圖
    -> 縮影 / 互動點 / 危險區 / 事件區
```

互動跳頁規則：

```text
接近物件
  -> 場上驚嘆號提示
    -> 點擊
      -> 進入管理介面
        -> 自動切到互動
          -> 自動切到對應設施子頁
```

## 9. 這份文件完成後的用途

這份骨架定下來之後，後續就可以很自然往下拆成：

- 管理介面線框稿
- 互動設施逐頁細稿
- 圖鑑頁細稿
- 地圖縮影頁細稿
- 各頁按鈕與文案規格

它的價值不在把所有細節一次說死，而是在先把管理系統的骨頭長正。
