# COrebound LAttence 主流程與頁面骨架統一版 (Master Flow & Page Skeleton)

本文件用途：把目前三份介面文件（[玩家介面_UserFlow方案.md](玩家介面_UserFlow方案.md)、[玩家介面_主畫面與管理介面補充規格.md](玩家介面_主畫面與管理介面補充規格.md)、[玩家介面_按鈕與操作總清單.md](玩家介面_按鈕與操作總清單.md)）以及 [遊戲設計文件.md](../遊戲總覽/遊戲設計文件.md) 中的 UI 段落收斂成**單一、不互相矛盾**的主流程與頁面骨架。

本文件不取代前三份文件的細節內容（動效描述、按鈕文字語氣等仍以原文件為準），但**若本文件與前三份文件在「結構、進出規則、頁面歸屬」上有衝突，以本文件為最終裁定版本**。

- 這是「階段一：方向定版與基礎骨架」的收斂交付物（對應 [開發進度清單.md](../遊戲總覽/開發進度清單.md) 中「文件之間不再互相矛盾」這個必須完成項目）。

## 1. 先處理：既有文件間的矛盾與裁定

在收斂骨架之前，三份介面文件與 GDD 之間存在下列結構性矛盾。裁定結果會直接反映在後面的流程圖與規則表中。

### 1.1 主畫面主按鈕數量與名稱

- **矛盾**：[玩家介面_UserFlow方案.md](玩家介面_UserFlow方案.md) 第 6.1 節定案為 5 個主按鈕（`開始遊玩 / 圖鑑 / 遊玩記錄 / 新手入門 / 設定`，離開遊戲不算主按鈕）。但 [玩家介面_按鈕與操作總清單.md](玩家介面_按鈕與操作總清單.md) 第 3.8 節仍列出一組較舊的清單（`開始對戰 / 編排小隊 / 圖鑑 / 設定 / 玩家資料 / 離開`）。
- **裁定**：採用 **5 主按鈕版本**為唯一正式版本。
  - `編排小隊` 不獨立設主按鈕，改為「New Game 準備流程」與「局內管理介面 / 小隊分頁」共用同一套小隊圓盤元件（見 2.1、3.2）。
  - `玩家資料` 併入 `遊玩記錄`（成就、歷史配置本來就是玩家資料的一部分）。
  - `離開` 維持原裁定：不算主按鈕，放置於角落次級位置。
  - `按鈕與操作總清單.md` 第 3.8 節內容視為**待併版舊稿**，之後應同步更新，本文件不重寫該檔。

### 1.2 圓盤三色語意（保護 / 支援 / 火力 / 補給）

- **矛盾**：GDD 2.3.3 用 `綠=保護、黃=支援、紅=火力`；[玩家介面_主畫面與管理介面補充規格.md](玩家介面_主畫面與管理介面補充規格.md) 第 2.4 節改用 `藍=保護、紅=火力、黃=補給`，且該文件明確聲明「這是你最新的說法，之後以此為準」。
- **裁定**：UI 層一律採用**補充規格版本**：`藍色＝保護位、紅色＝火力位、黃色＝補給位`。
  - GDD 的 `綠 / 支援` 屬於尚未同步的舊用語，本文件不修改 GDD 本體，但所有頁面骨架、按鈕、圖例一律以「藍 / 紅 / 黃、保護 / 火力 / 補給」為準。
  - 建議另開任務同步修正 GDD 用語（不在本次骨架文件範圍內）。

### 1.3 圖鑑的兩個入口是否算同一頁面

- **矛盾**：`圖鑑` 同時是主畫面第一層主按鈕的子項（[UserFlow方案.md](玩家介面_UserFlow方案.md) 6.2 節，7 個子項，含「世界故事」），也是局內管理介面的第一層主資料夾（[補充規格.md](玩家介面_主畫面與管理介面補充規格.md) 5.8、5.12 節，6 個子項，**不含**「世界故事」）。兩份文件沒有明講這是「同一個元件兩個入口」還是「兩個獨立系統」。
- **裁定**：**是同一套圖鑑 UI 元件，兩個掛載點**，但子項清單刻意不同：
  - 主畫面版（戰鬥外）：`成員圖鑑 / 怪物圖鑑 / 世界圖鑑 / 材料圖鑑 / 機制圖鑑 / Boss圖鑑 / 世界故事`（7 項）。
  - 局內版（戰鬥中，掛在管理介面）：`成員圖鑑 / 怪物圖鑑 / 世界圖鑑 / 材料圖鑑 / 機制圖鑑 / Boss圖鑑`（6 項，**不含世界故事**）。
  - **理由**：世界故事屬於「跨局、帳號等級的輪迴敘事解鎖」，一輪一輪逐步揭露（見 [世界故事_輪迴敘事與四隊長人物設計.md](../世界觀/世界故事_輪迴敘事與四隊長人物設計.md)），節奏設計是給玩家在對局外沉浸閱讀，戰鬥中局勢緊繃時不適合被推銷劇情，因此局內管理介面刻意拿掉這一項。
  - 資料查詢類（成員/怪物/世界/材料/機制/Boss）兩邊完全共用同一份資料與同一個元件版面（左側書籤／中央內容／右側補充資訊）。

### 1.4 地圖是否屬於圖鑑

- 兩份文件已有共識：`地圖`（縮影/互動點/危險區/事件區）**不屬於圖鑑**，是管理介面第一層獨立主資料夾，且只存在於戰鬥中（沒有數據可看時，戰鬥外沒有地圖頁）。此點無矛盾，本文件直接沿用。

### 1.5 結算頁在既有文件中缺乏流程定位

- **缺口**：[玩家介面_按鈕與操作總清單.md](玩家介面_按鈕與操作總清單.md) 3.7 節列出結算頁按鈕（`回大廳 / 再來一場 / 查看戰報 / ...`），但 [UserFlow方案.md](玩家介面_UserFlow方案.md) 第 12 節的完整流程圖完全沒有畫出結算頁，也沒說它何時出現、離開後去哪。
- **裁定**：補上結算頁為正式遊玩流程的終點頁，規則見 3.9、5 的 R10、R11。

### 2.2 訓練道場的特殊定位

`新手入門 → 訓練道場` 在主畫面內觸發，但**不是獨立 UI**：它直接借用「正式遊玩」的操作頁面 + 管理介面整套骨架，只是資料源換成沙盒配置（怪物可自由召喚、無縮圈、無死亡懲罰）。退出訓練道場固定回到主畫面，不會出現結算頁（沒有勝負判定）。

## 4. 戰鬥中 / 戰鬥外頁面分類

### 4.1 純戰鬥外 (OOC only)

- 主畫面首頁
- 開始遊玩子選單（New Game / Continue Game / 多人連線）
- 圖鑑（主畫面版，含世界故事）
- 遊玩記錄（存檔/通關/成就等，帳號等級資料，對局內無對應頁）
- 新手導覽
- 設定
- 結算頁的「回大廳」落點（結算頁本身視為 IC→OOC 過渡頁，見 4.3）

### 4.2 純戰鬥中 (IC only)

- 操作頁面（戰鬥 HUD）及其左滑技能列／右滑物品欄
- 管理介面／背包
- 管理介面／互動（合成/熔爐/雕像/商店/召喚）
- 管理介面／地圖

### 4.3 跨情境 / 過渡頁面

| 頁面 | 說明 |
|---|---|
| 圖鑑（資料查詢類 6 項） | 共用元件，主畫面與管理介面各掛一份，內容完全相同 |
| 小隊（大圓盤＋立繪） | 元件在 New Game 準備流程（賽前預覽）與管理介面（賽中管理）共用，但只有管理介面版可即時調整戰鬥用陣型 |
| 訓練道場 | 掛載點在主畫面，內部骨架借用 IC 的操作頁面＋管理介面 |
| 結算頁 | 由 IC 的強制事件觸發，落點通向 OOC（回大廳）或直接重進 IC 準備流程（再來一場） |

## 5. 衝突規則清單 (Conflict / Precedence Rules)

以下規則用來解決「兩個以上介面狀態同時想要生效」時，誰贏、誰讓步。

- **R1．左右滑動互斥**：操作頁面的左滑技能列與右滑物品欄同一時間只能開一邊；開啟其中一邊時，另一邊的「可滑動」提示也一併隱藏，不可疊加顯示。
- **R2．管理介面與戰鬥 HUD 展開層互斥**：一旦進入管理介面，操作頁面的左滑／右滑狀態必須自動收回並鎖定；同一時間畫面上只允許存在「操作頁面層」或「管理介面層」其中一層。
- **R3．管理介面不暫停世界時間**：本作是即時多人大逃殺，管理介面只是本地 UI 覆蓋層，縮圈侵蝕、其他玩家行動、怪物 AI、能量自然恢復等世界時間持續運作。UI 需在管理介面中提供最低限度的危險提示（例如縮圈剩餘時間），避免玩家因長時間停留管理介面而不自覺陣亡。
- **R4．小隊賽前配置缺口**：目前 5 主按鈕主畫面沒有獨立「編排小隊」入口（見 1.1 裁定），賽前小隊調整只能發生在 New Game 準備流程。若未來要支援「開局前先手動精調全部圈層位置」，需在 New Game 準備流程內掛載完整的「小隊」分頁（共用元件 B），而不是新增主畫面按鈕。
- **R5．互動子分頁各自獨立鎖定**：管理介面／互動下的合成、熔爐、雕像、商店、召喚，各自依「玩家目前是否正靠近對應設施」獨立判定啟用狀態；打開其中一個分頁不影響其他分頁的鎖定狀態，彼此互不連動。
- **R6．驚嘆號跳轉優先權高於手動瀏覽**：玩家透過驚嘆號提示開啟管理介面時，必須直接跳到互動下的對應子分頁；即使玩家先前手動瀏覽到別的分頁（例如圖鑑），下一次驚嘆號觸發仍會覆蓋跳轉到對應互動子分頁，不保留玩家原本停留的分頁。
- **R7．管理介面只能回操作頁面，不能直接回主畫面**：「返回戰場」按鈕（或再次觸發收合圓盤的操作）一律回到操作頁面；管理介面沒有任何路徑可以直接跳回主畫面，回主畫面只能經過結算頁的「回大廳」。
- **R8．圖鑑雙掛載點內容差異固定**：主畫面版圖鑑固定比管理介面版多「世界故事」一項；除此之外兩邊共用同一份資料與同一套版面元件，不允許各自維護不同的圖鑑資料源。
- **R9．遊玩記錄 / 設定不進入對局**：這兩個主按鈕的內容屬於帳號等級資料，管理介面不掛載對應分頁；如果玩家在對局中需要調整音量等設定，應視為系統層級快捷鍵（例如 ESC 選單），不佔用管理介面的 5 個第一層資料夾名額。
- **R10．結算頁的兩個出口互斥且各自導向不同流程**：「回大廳」只能導向主畫面；「再來一場」只能直接導向 New Game 準備流程；兩者不可能同時觸發，且都不能返回剛結束的那一局。
- **R11．死亡／終局事件優先權最高**：隊長死亡、對局勝利、斷線等終局事件，可以強制打斷「操作頁面」的左右滑狀態、以及「管理介面」正在瀏覽的任何分頁，直接跳轉結算頁；此優先權高於本清單所有其他規則（R1、R2、R6 等在終局事件觸發當下一律失效）。
- **R12．訓練道場複用 IC 骨架但不觸發 R10／R11**：訓練道場借用操作頁面與管理介面，但因為沒有勝負判定，不會觸發終局事件與結算頁；退出方式是玩家主動操作（例如返回鍵），直接回主畫面。
